【黑果小兵】【微信首发】macOS Big Sur 11.2.2 20D80 Installer for OpenCore 0.6.7 and CLOVER 5131 and PE三EFI分区原版镜像

3月1日，苹果发布了Big Sur的11.2.2正式版，版本号为20D80，新版本同时支持INTEL/ARM平台，未来几年内会渐渐取消对INTEL平台的支持，所以黑苹果的FANS务必要且行且珍惜

macOS Big Sur 将强大实力和优美外观的结合提升到一个崭新的高度。精心雕琢的全新设计，让您能淋漓尽致地感受 Mac 的魅力；Safari 浏览器迎来重大更新，待您饱览；地图 App 和信息 App 满载新功能，任您探索；更透明的隐私权限，保护也更周到。

版本: macOS Big Sur 11.2.2 20D80

发行日期: 2021年3月1日

苹果已经发布了macOS Big Sur 11.2.2。面向MacBook Pro和MacBook Air用户的修复程序今天向公众发布。苹果公司表示，该更新“防止”某些型号连接到第三方USB-C集线器和扩展坞时遭受“损坏”。

Apple解释说，与“第三方不兼容的有源USB-C”配件一起使用时，此更新可防止2019年或之后的MacBook Pro和2020年或以后的MacBook Air型号受到损坏。

完整的发行说明说明：

当MacOS Big Sur 11.2.2连接到某些第三方的，不兼容的有源USB-C集线器和扩展坞时，可以防止MacBook Pro（2019或更高版本）和MacBook Air（2020或更高版本）型号受到损坏。

不幸的是，Apple没有提供有关此bug的其他信息，因此尚不清楚某些Mac用户在使用第三方USB-C集线器和扩展坞时是否经历了“损坏”。

现在，您可以转到“系统偏好设置”应用程序中的“软件更新”菜单，将Mac更新到最新版本的macOS Big Sur。此更新适用于所有macOS Big Sur用户，并且不仅限于受影响的计算机。

镜像特点：

  • 三个独立的EFI引导分区，同时支持 OpenCore / CLOVER / WEPE引导
  • 从本版开始，默认主EFI分区为OpenCore引导，挂载后的卷标为：OC，第二个EFI分区为CLOVER，挂载的卷标为：CLOVER
  • 新增全平台配置文件，请根据自己的架构选择食用：
    • config 默认300/400系列主板
    • config_BCM94360Z4.plist 新增BCM94360Z4网卡驱动的配置文件
    • config_SandyBridge
    • config_IvyBridge
    • config_SNB_and_IVB HEDT
    • config_Haswell
    • config_Haswell-HEDT
    • config_Broadwell
    • config_Broadwell-HEDT
    • config_SkyLake
    • config_CascadeLake_and_SkyLake-HEDT
    • config_KabyLake
    • config_CoffeeLake
    • config_CometLake
    • config.orig 备用
  • 同时支持 Intel 及 AMD 双平台安装使用，RYZEN CPU请选择 config_AMD_RYZEN 进行安装
  • PE 可连接网络，可远程协助，集成向日葵 和 AnyDesk
  • 附带exFAT分区，用于装入Windows 10等的iso安装镜像，可直接安装双系统
  • 支持UEFI / MBR引导，基本上做到一盘在手，可以直接引导安装某些不支持UEFI引导方式的机型，比如Dell inspiron 7559等。

OC 分区

容量：200MB

  • 安装过程中默认会禁用博通的无线网卡，请安装结束后移除引导参数brcmfx-driver=0以使用博通无线网卡

BIOS设置选项列表

禁用清单

  • Fast Boot - 快速启动
  • VT-d (can be enabled if you set DisableIoMapper to YES) - VT-d（如果DisableIOMapper Quicks设置为YES，则可以启用）
  • CSM - CSM 兼容性支持模块
  • Thunderbolt - 雷雳
  • Intel SGX - 英特尔SGX
  • Intel Platform Trust- 英特尔平台信任
  • CFG Lock (MSR 0xE2 write protection) - CFG锁（MSR 0xE2写保护）（必须关闭，如果找不到该选项，则在OpenCore的config-内核-> Quirks下启用与CFG Lock相关选项）
  • ecure Boot - 安全启动
  • Parallel Port - 并口
  • Serial/COM Port - 串行/COM端口

启用

  • VT-x - VT-x
  • UEFI Boot Mode UEFI启动模式。请不要使用Legacy
  • 硬盘模式：改为AHCI。不能用IDE和RST RAID。
  • Above 4G decoding - 大于4G地址空间解码
  • Hyper-Threading - 超线程
  • Execute Disable Bit - 执行禁用位
  • EHCI/XHCI Hand-off - EHCI / XHCI接手控制
  • OS type: Windows 8.1/10 UEFI Mode - 操作系统类型：Windows 8.1 / 10 UEFI模式
  • DVMT Pre-Allocated(iGPU Memory): DVMT预分配（iGPU内存）：64MB（如果能设Max就设）
  • Legacy RTC Device - 传统RTC设备
